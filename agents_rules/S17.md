### 17. 模块级 `let`/类 `private` 可变属性访问需用 `!!`（smart cast 失效）
**规则**：
- UTS 编译器对**模块级 `let` 变量**和**类 `private` 可变属性**不做 smart cast
- 即使在 `if (x != null)` 块内、`x != null && x.method()` 短路中、或是三元表达式中，都需要显式 `!!`
- **必须用 `!!` 模式**：
  ```uts
  if (service != null) { service!!.setCallbacks(c) }
  if (this.handler != null) { this.handler!!.postDelayed(...) }
  return service != null ? service!!.getInfo() : null
  return service != null && service!!.isConnected()
  ```
- **替代方案**：先 `const x = this.handler; if (x != null) x.method()`（局部 val 可触发 smart cast）
- 重要：该规则与 Kotlin 类似，但 UTS 编译器更严格，连 `if (x != null)` 块内都不做 smart cast

### 17.b 补充：`const x = props.obj` 推断为 nullable 后，`x.field` 与 `x!!.field` 都报 error18
**触发场景**（与上面 17 不同）：
- `const r = props.suggestedRule`（props 类型为 `EffectiveTriggerRule | null`），r 推断为 `EffectiveTriggerRule | null`
- `if (r == null) return ''` 后访问 `r.actionId` 报 **"找不到名称 'actionId'"**（不是 null safety 错误，是类型查找失败）
- 改用 `r!!.actionId` **同样报"找不到名称"**——`r!!` 在 UTS 属性访问位置不工作（仅 `x!!.method()` 函数调用位置 work）

**唯一可靠的修复**：抽函数 + 形参具体类型 + 调用前 `as` 断言
```uts
function actionNameOf(r: EffectiveTriggerRule): string {  // 形参非空
  const a = getActionById(r.actionId)
  return a != null ? a.name : r.actionId
}

const actionName = computed<string>((): string => {
  const r = props.suggestedRule
  if (r == null) return ''
  return actionNameOf(r as EffectiveTriggerRule)  // r 推断为 nullable, as 断言
})
```
- 抽函数形参用具体非空类型（如 `EffectiveTriggerRule`），函数体内访问 `.field` 不会触发 narrow 失败（参考 `components/LlmHistoryCard.uvue` 的 `formatRule` 模式）
- `r as EffectiveTriggerRule` 是窄口，调用方已 `if (r == null) return ''` 兜底，不会运行时崩
- 备选：把抽函数的形参也声明为 nullable，函数体内 `if (r == null) return ''` + 再访问——但若函数体仍不 narrow 会再次遇到此问题，不推荐

**根因推测**：UTS 编译器对 `const r = props.x`（局部 const 但来源是 props）保留为精确联合类型，对 `x!!.field` 这种位置不识别为 narrow 操作。
